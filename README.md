# 🧩 IT-Ребус Челлендж

Соревновательная веб-платформа для **ИТ-каникул / проф-ориентации** по специальности
«Разработка и управление программным обеспечением». 7 этапов, очки, живой рейтинг,
достижения и отдельный **экран ведущего** для проектора.

Технологии: **Node.js + Express + MySQL** (без нативной сборки — `mysql2`).

---

## 🎮 Что внутри

| Этап | Описание |
|------|----------|
| 🧩 Эмодзи-ребус | Расшифруй IT-слово по эмодзи |
| 💻 Допиши код | Подставь нужный кусочек (с подсказками) |
| 🎮 Угадай игру | Какая игра спряталась в описании |
| ⭕ Крестики-нолики | Игра вживую с другим студентом, а если соперников нет — с ботом (открывается после 2 этапов) |
| 🔮 Что выведет код | Угадай результат строчки кода |
| 🟩 IT-Wordle | Угадай слово из 5 букв за 6 попыток |
| ⚡ Правда или миф | Блиц по фактам из мира технологий |

- **Главная страница** с кнопкой **«Демо для проектора»** — примеры всех этапов, чтобы объяснить правила.
- **Экран ведущего** (`/admin`) — живой дашборд: онлайн, рейтинг, прогресс по этапам, лента достижений.

## 🚀 Как запустить на Railway (по шагам)

### 1. Залить код на GitHub
В этой папке выполни:

```bash
git init
git add .
git commit -m "IT-Ребус Челлендж"
git branch -M main
git remote add origin https://github.com/ds23432c/kit-sistem.git
git push -u origin main
```

### 2. Создать сервис из репозитория
В проекте Railway (там же, где уже подключён MySQL):
**New → GitHub Repo → kit-sistem**. Railway сам поймёт, что это Node.js, и выполнит `npm install` и `npm start`.

### 3. Подключить базу к приложению (важно!)
Открой сервис приложения → вкладка **Variables** → **New Variable → Add Reference** и добавь:

| Переменная | Значение (Reference) |
|------------|----------------------|
| `MYSQL_URL` | `${{MySQL.MYSQL_URL}}` |

> Это «приватный» адрес базы (`mysql.railway.internal`) — приложение и база общаются внутри Railway.
> Приложение само читает `MYSQL_URL`. Если её нет — берёт `MYSQLHOST / MYSQLUSER / MYSQLPASSWORD / MYSQLDATABASE / MYSQLPORT`.

### 4. Задать логин/пароль ведущего (рекомендуется)
В тех же Variables приложения добавь:

| Переменная | Значение |
|------------|----------|
| `ADMIN_LOGIN` | твой логин |
| `ADMIN_PASSWORD` | твой пароль |

Если не задать — по умолчанию **admin / admin1234** (потом поменяй).

### 5. Готово
Railway выдаст ссылку (Settings → Networking → **Generate Domain**).
- Студенты открывают ссылку, регистрируются (имя + пароль) и играют.
- Ты открываешь `https://тот-же-адрес/admin`, входишь под админ-аккаунтом → **экран ведущего**.

`PORT` Railway подставляет автоматически — менять ничего не нужно.

## 🖥 Как провести занятие

1. Открой главную → **«Демо для проектора»**, покажи примеры этапов (объясни правила).
2. Дай ребятам ссылку (можно показать QR-код адреса на экране).
3. Открой `/admin` на проекторе — все видят живой рейтинг и прогресс.
4. Через пару минут после старта откроются крестики-нолики (нужно пройти 2 этапа) — ребята играют друг с другом.

## ⚙️ Локальный запуск (по желанию)

Нужен установленный MySQL. Затем:

```bash
npm install
MYSQL_URL="mysql://root:ПАРОЛЬ@127.0.0.1:3306/railway" npm start
```

Открой http://localhost:3000

## 🏆 Очки

- Правильный ответ — **+100**
- Пройденный этап — **+50** бонусом
- IT-Wordle — чем быстрее угадал, тем больше (до **+600**)
- Крестики-нолики: победа **+300**, ничья **+150**, поражение **+50**

Достижения открываются за прохождение этапов, идеальные ответы, победы и лидерство в рейтинге.
